What is this?

GPU Finder is a minimal, client‑side UI for the read‑only endpoint GET https://gpu-api-891435322100.us-central1.run.app/gpus. It renders the results as returned—no hidden normalization.

Filtering & Sorting

All filters are passed via query parameters. Supported parameters:

ParamTypeMeaningExample
sourcestringProvider (e.g., vastai, tensordock, runpod)?source=runpod
locationstringCase‑insensitive substring match?location=us-west
max_pricenumberMax total_cost_ph ($ per hour)?max_price=2.5
min_flopsdnumberMin flops_per_dollar_ph?min_flopsd=10
sortstringcolumn.direction (default updated_at.desc)?sort=total_cost_ph.asc
limitinteger1–1000 (default 200)?limit=200
offsetintegerPagination offset?offset=200

Fields

Each item is a GPU object with the fields below (subset shown):

FieldTypeNotes
idstringInstance identifier
sourcestringProvider name
locationstringRegion/zone (free‑text)
namestringGPU model label as provided
num_gpusintegerCount of GPUs
vram_mbintegerVRAM in MB (UI shows MB plus an approximate GB conversion MB/1024)
ram_mbintegerSystem RAM in MB (UI shows MB and ~GB)
total_flopsnumberNumeric FLOPs value as supplied; unit/precision may vary by provider
flops_per_dollar_phnumberFLOPs per $ per hour
total_cost_phnumberTotal $ per hour
reliabilitynumber0–1, displayed as %
upload_mbps, download_mbpsnumberNetwork throughput
urlstringLink to provider page for the offer
updated_atstringISO timestamp

Notes on correctness

  • The UI does not reinterpret metrics; it displays fields as returned. Any provider‑specific meaning (e.g., FLOPs precision) is preserved.
  • VRAM/RAM are shown in MB with a convenience (~GB) conversion using MB/1024.
  • Sorting is delegated to the API via sort whenever possible.
